Hermes Agent Local Architecture

local/Discord 환경에서 운영 중인 Hermes Agent의 profile, Docker backend, GitHub 인증, Virtual Main sync, Dashboard, 문서 저장소의 관계를 한눈에 보기 위한 아키텍처 문서입니다.

Project: shaul-hub/hermes-agent-local-docs Scope: local operations docs Not a Hermes Agent source fork

1. 아키텍처 요약

이 저장소는 Hermes Dashboard나 Hermes Agent 소스코드를 대체하지 않습니다. Dashboard는 실행 중인 agent/gateway 상태를 관찰하는 UI이고, 이 저장소는 local 운영 지식, 커스텀 결정, 복구 절차, 장애 이력을 보존하는 문서 source-of-truth입니다.

draw.io에서 편집 가능한 원본은 architecture.drawio에 보관합니다.

Runtime layer
Discord hub bot, Hermes gateway, provider, tools, Docker terminal backend가 실제 작업을 수행합니다.
Operations layer
GitHub 인증, repo push, Virtual Main sync, profile config, dashboard 관찰이 운영 대상입니다.
Documentation layer
이 repo는 운영 원칙, customization, runbook, incident 기록을 관리합니다.

Hermes runtime 인증/운영 경계 source-of-truth / repo 사용자-visible 검증 대상

3. 핵심 경계: Host와 Docker Container

hub profile의 terminal backend는 Docker 컨테이너에서 실행됩니다. 따라서 host의 파일, 인증, 환경변수, Hermes profile 설정이 컨테이너 안에서 자동으로 보이지 않습니다. GitHub repo 작업 실패의 대표 원인은 repo 상태가 아니라 gh 인증 config가 container에 없거나 다른 경로를 바라보는 경우입니다.

항목 Host 기준 Container 기준 운영 의미
GitHub 인증 /Users/jihoonkim/.hermes/profiles/hub/gh $GH_CONFIG_DIR, 현재 /root/.config/gh repo create/push 전 container의 gh auth status가 기준입니다.
작업 경로 local project path /workspace 또는 worktree path 작업 전 git rev-parse --show-toplevel로 확인합니다.
설정 동기화 Virtual Main mirror/symlink mount된 파일만 접근 가능 host source-of-truth와 container runtime view를 구분합니다.

4. GitHub 인증 및 Push 흐름

Push 전 확인 명령

export PATH=/workspace/bin:$PATH
export HOME=/root
export GH_CONFIG_DIR=/root/.config/gh

gh auth status
git status --short --branch
git remote -v

Push 후 검증 명령

local=$(git rev-parse HEAD)
remote=$(gh api repos/shaul-hub/hermes-agent-local-docs/git/ref/heads/main --jq '.object.sha')
printf 'local=%s\nremote=%s\n' "$local" "$remote"
test "$local" = "$remote"

6. Discord Delivery 검증 경로

Discord long-response/attachment 문제는 사용자-visible delivery path에서 검증합니다. 내부 도구 호출 성공만으로 완료 처리하지 않습니다.

7. 운영 Checklist

  • 작업 시작 전 repo root, branch, remote, auth status를 확인합니다.
  • GitHub 작업은 container 기준 gh auth status가 성공할 때만 진행합니다.
  • host auth와 container auth를 같은 것으로 가정하지 않습니다.
  • secret/token 값은 문서나 commit에 포함하지 않습니다.
  • Dashboard는 runtime UI, 이 repo는 운영 문서 source-of-truth로 분리합니다.
  • Discord delivery 이슈는 실제 사용자-visible 결과로 검증합니다.
