# Research Notes

## 조사 요약

AI Resource Platform은 단순 문서 저장소가 아니라, 사내 AI 활용의 탐색성·신뢰성·거버넌스를 함께 해결해야 합니다. 핵심은 `리소스 카탈로그`, `승인/정책 상태`, `재사용 가능한 템플릿`, `운영 워크플로`입니다.

## 사용자 니즈

### 일반 사용자

- 내 업무에 바로 쓸 수 있는 AI 도구를 알고 싶다.
- 검증된 프롬프트와 예시를 복사해 쓰고 싶다.
- 어떤 데이터를 넣으면 안 되는지 명확히 알고 싶다.

### 운영자

- 흩어진 AI 자료를 정리하고 최신 상태로 유지하고 싶다.
- 승인되지 않은 도구 사용을 줄이고 싶다.
- 좋은 활용 사례를 확산하고 싶다.

### 보안/컴플라이언스

- 민감정보 입력 금지, 외부 도구 사용 제한 등 기준을 명확히 전달하고 싶다.
- 도구별 위험도와 승인 상태를 관리하고 싶다.

## 경쟁/대체 수단

- 사내 위키: 문서화에는 좋지만 구조화된 검색/상태/검토 흐름이 약하다.
- 스프레드시트: 초기 목록 관리에는 빠르지만 확장성과 검토 이력이 약하다.
- LMS: 학습에는 좋지만 도구/프롬프트 카탈로그에는 부적합하다.
- Slack/Discord 채널: 질문 응답에는 좋지만 지식 누적과 최신성 관리가 어렵다.

## MVP 제안

1. 정적/문서 기반으로 시작한다.
2. 리소스 메타데이터 스키마를 먼저 정의한다.
3. 검색 가능한 카탈로그를 만든다.
4. 승인 상태와 정책 링크를 필수 필드로 둔다.
5. 신규 리소스 제안 템플릿과 검토 체크리스트를 둔다.

## 리스크

- 리소스 owner가 불명확하면 문서가 빠르게 낡는다.
- 정책 표현이 모호하면 사용자가 안전하지 않은 도구를 계속 사용한다.
- 검색/태그 설계가 약하면 플랫폼이 또 다른 문서 저장소가 된다.
- 기여 흐름이 복잡하면 리소스 등록이 줄어든다.
