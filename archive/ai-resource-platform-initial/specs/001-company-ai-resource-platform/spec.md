# Spec: Company AI Resource Platform MVP

## 목적

사내 AI 관련 리소스를 중앙에서 탐색하고, 검토된 자료를 안전하게 재사용할 수 있게 한다.

## 사용자 시나리오

### US-001: 리소스 탐색

구성원으로서 나는 업무 목적에 맞는 AI 도구와 템플릿을 빠르게 찾고 싶다.

#### Acceptance Criteria

- 리소스는 유형, 태그, 승인 상태, owner 기준으로 구분된다.
- 사용자는 제목/설명/태그 기반으로 리소스를 찾을 수 있다.
- 리소스 상세에는 사용 조건과 정책 링크가 표시된다.

### US-002: 프롬프트 템플릿 재사용

구성원으로서 나는 검증된 프롬프트를 복사해 내 업무에 적용하고 싶다.

#### Acceptance Criteria

- 프롬프트 템플릿에는 목적, 입력값, 출력 기대 형식, 주의 사항이 포함된다.
- 템플릿에는 owner와 최신 검토일이 표시된다.

### US-003: 리소스 등록/검토

운영자로서 나는 신규 리소스를 제안받아 검토 후 카탈로그에 노출하고 싶다.

#### Acceptance Criteria

- 제안자는 리소스 유형, 설명, owner, 정책 영향 정보를 입력한다.
- reviewer는 승인/수정 요청/반려 상태를 기록할 수 있다.
- 승인된 리소스만 기본 카탈로그에 노출된다.

### US-004: 정책 확인

사용자로서 나는 도구 사용 전에 데이터 취급 기준과 제한 사항을 확인하고 싶다.

#### Acceptance Criteria

- 도구 상세에는 승인 상태가 표시된다.
- 민감정보, 고객정보, 소스코드 등 데이터 유형별 주의 사항을 연결한다.

## 기능 요구사항

1. 리소스 메타데이터 스키마 정의
2. 리소스 카탈로그 문서/페이지 구조 정의
3. 정책/가이드 문서와 리소스 연결
4. 신규 리소스 제안 템플릿 제공
5. 검토 체크리스트 제공

## 비기능 요구사항

- 정보 최신성: owner와 검토일 필수
- 보안성: 승인 상태와 데이터 취급 기준 필수
- 확장성: 리소스 유형 추가 가능
- 운영성: 간단한 문서 기반 프로세스로 시작 가능
