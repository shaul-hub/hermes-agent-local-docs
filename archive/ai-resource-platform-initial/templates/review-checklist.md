# Review Checklist

## 기본 정보

- [ ] 제목과 요약이 명확하다.
- [ ] owner가 지정되어 있다.
- [ ] 리소스 유형이 적절하다.
- [ ] 최신 검토일이 있다.

## 품질

- [ ] 사용 목적이 명확하다.
- [ ] 사용 방법이 재현 가능하다.
- [ ] 예시 또는 샘플이 충분하다.
- [ ] 중복 리소스가 아닌지 확인했다.

## 보안/정책

- [ ] 승인 상태를 지정했다.
- [ ] 민감정보/고객정보/소스코드 입력 가능 여부를 검토했다.
- [ ] 관련 정책 문서를 연결했다.
- [ ] 제한 사항과 주의 사항을 명확히 적었다.

## 운영

- [ ] owner가 유지보수 가능하다.
- [ ] 만료/재검토 기준이 있다.
- [ ] 사용자 피드백 수집 방법이 있다.
